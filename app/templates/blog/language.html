MOST USEFUL RESOURCES FOR LEARNING PYTHON

  • Adam Cribbs
  • April 5, 2017

One of the classic questions that is always asked by any aspiring computational biologist is, which language should I learn? I hear this question asked so many times that I thought I would share my opinion on the matter. Although I must say, I don’t really think there is a single right answer. Ultimately the language you learn has to be guided by the type of work you will be expected to be performing. I will try and layout the advantages and disadvantages of some of the most useful languages in the following paragraphs.

As a novice, choosing the right language can seem a daunting task. For instance, wikipedia collects a list of all historical and active programming languages, looking at the list and roughly counting them I see over 500!. This seems like a huge number, however there are only really a few that I would consider as a computational biologist. My shortlist would be bash, Python, R, Perl, C++ or Julia (although I have less experience with Julia but I can see big things for this language). So lets set out the advantages and disadvantages of learning each of these programming languages for computational biology:

R:

Advantages:
  • Open source and a thriving community of developers for bioinformatics.
  • Well-maintained packages and good package managers.
  • Was designed as a statistical language, therefore it deals really well with dataframes and mathematic operations.
Disadvantages:
  • Its not a very quick language and it can be quite a difficult language to learn as a new user.
  • Personally I find the syntax confusing, particularly when other users embed functions within functions on the same line without separation). i.e. the language doesn’t really enforce any style (which for some that may be good, but for me it makes the code sometimes difficult to interpret)

Python:

Advantages:
  • Python works great as a general purpose tool
  • A huge advantage is its easy to read syntax and is a great language to start to learn as a beginner
Disadvantages:
  • Python is an interpreted language so if you need to perform lost of high performance computations this isn’t the language for you
  • Around 30-40X slower than C++

Perl:

Advantages:
  • Was the main language of choice for many years in computational biology and has received good support.
  • The regular expression ability of perl is unmatched when compared to any other programming languages.
  • Perl one liners are very useful and I have used them many times to as a quick way to manipulate files (http://www.catonmat.net/blog/introduction-to-perl-one-liners/).
Disadvantages:
  • While many computational biology projects are written in perl, its use has been declining in the past couple of years and python is taking over its place.
  • In my experience, there are fewer resources available to beginners to pick up this language, although there are a few good of videos on you tube.

C++:

Advantages:
  • Very fast
  • Can be used to process large files and perform mathematical operations with ease
  • Highly portable language
  • Has a very rich functional library
Disadvantages:
  • Learning and mastering this language can be difficult for a beginner
  • Its not used that frequently in computational biology because of the dominance of python and perl.

Julia:

Advantages:
  • This has a similar feel to python and is easy to pick up if you are already familiar with the syntax
  • It runs really fast (within 2X speed of C++) so if you find something is running very slow in python you can use Julia to overcome this.
  • Aims to be a high-level prototyping language that is also fast enough for simulation programming
Disadvantages:
  • It’s a relatively new language (2012) so the package development is at a relatively young stage, however there is a good community beginning to develop.

My language evolution:

To give some personal perspective, the first language I learnt was R then I moved onto python. However, Speaking with hindsight, I would have started with python as it’s a really good language to start to learn the fundamentals of programming. There are also so many more really good tutorials on python than R. The main reason for starting with R first was because I wanted the power to perform vectorised programming, because I was working with large data frames that needed to be sorted and manipulated. However, with the introduction of the pandas python module, all of this can now be performed in python. Most of R functionality can now be found in python, however R still has one major advantage and that’s the number of packages developed for Next generation sequencing in R. This makes R approachable to users that are not necessarily computationally literate.

Most computational biologists can stop when they learn R and python as they are versatile enough to give you the ability to process, analyse and plot most data that you will encounter. However, if you really want to streamline your code and write efficient computational pipelines you may need to learn a few other languages to achieve this.

One of the most useful for wrapping code into command line functions is bash scripting. Before I joined CGAT most of the code I wrote to run programs on the commandline was by writing bash scripts. Moreover, it’s a really useful language to learn if you want to run programs on multiple files in a directory.

The most difficult language Iv learnt was probably C++. I pushed myself to learn it because I wanted to understand some of the source code written by google in their open source Tensorflow project . Although I don’t often write C++ code, it’s a great language to learn as it gives you experience with some pretty cool concepts in programming such as switches and teaches you about how computers hold things in memory. Similarly, I began to learn the Julia language, not out of necessity but because I was interested in seeing if it could possibly be useful in the future to speed up code I was writing in python. Although I don’t use it that often at the moment because in most cases python is sufficient, I do see a big future for this language as its aim is to be a high-level language that runs within 2X the speed of C++ (In certain benchmarking its faster!).
