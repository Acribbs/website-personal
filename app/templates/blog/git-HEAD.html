GIT - HOW TO ATTACH A DETACHED HEAD

  • Adam Cribbs
  • March 29, 2017

I love git as a version control system. I find it easy to use and integration with github makes it a great way to share and contibute to code projects

However, I encountered a problem recently that I hadn't seen before (see the title), the problem was caused by my lack of knowledge, some of you may be aware of how to fix this, some of you not. It certainly took me a while to solve so i will share with you the problem and how to overcome it.

My problem was caused when I wanted to revert back to a previous commit, look at that commit, and then revert back to the HEAD. I will use an example to demonstrate:

I first ran the command: 

git log

Then I reverted back to a previous commit

git checkout 0927d59326e225b408c89c11c6f7479bff04ac77

You can clearly see now that the HEAD is at 0927d59. What I wanted to do now was simple, move back to the HEAD

I tried the following without success:

git checkout 0927d59
git checkout HEAD

I was completely stuck, however after about 20 mins of trying any and all commands, it should have been obvious. I ran:

git checkout master

This reset the HEAD back to the master and I was clear to continue working on the code.
