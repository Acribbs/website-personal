BASIC GITHUB TUTORIAL

  • Adam Cribbs
  • March 29, 2017

In this post I would like to discuss the process that you go through to use github to set up a repository, clone to your local machine to add files or changes then push the changes to github

What is git and github?

GitHub is a code hosting web platform for version control and collaboration. It lets you and others work together on projects from anywhere.

git is the command line interface that allows you to perform version controlling on your local machine and push changes to a remote repository (i.e. interact with github.com). I have written a previous blog post on references to which tutorials I found most useful here.

Sign up for a github account

When you go to the website https://github.com you should see a signup page as shown below. Sign up for an account and login

When you login you should see a user page. You can add a picture and change settings of your account by clicking the top right image and selecting settings. The user page displays information regarding your code contributions, repositories, starred repositories and followers.

One of the main benefits to using github is to organise your code into repositories. This makes contributing to your code and other peoples code much easier. To set up a new repository select the repository tab then click the green new button. You can then give it a name and select create. This will create a blank repository that you can clone onto your local machine and start to create version controlled code.

Once you have made your respiratory, next you need to clone it so you can start working on it. To do this you copy the http link that appears in the code section of the repository. Then you follow the instructions below:

The first thing you may want to do is to add files to your repository and start to version control your files. Below you can see that I have added the file empty.txt to the cloned repository. I then run git status and notice that this file is untracked. At this point, Git is aware that you created a new file, but the file is untracked, meaning that git does not see it as part of your repository commits. The next thing I need to do is to track this file so git has a record of its state. Once a file is tracked this will allow you to track file changes.

To tell git to add and track your file in the repository you need to run the git add empty.txt command. Then you can run git status, below you can see the output of running this command

You can see from the output in green that the file is ready to be committed. To commit the changes run git commit -m "Insert a message to remind you of the changes that were made". The -m option stores a commit message in the log. Next, when you run git status you will see that the empty.txt file is no longer ready to be committed and the commit message is empty.

So when you have finished making changes on your local branch (i.e. on you laptop) you now need to push your code to the remote repository (i.e. github). This is achieved by running the command git push.

This completes the first introduction to github, in future posts I intend on writing posts detailing how to contribute to cgat code and also how to use some of git's more advanced tracking features
