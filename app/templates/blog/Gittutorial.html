VERSION CONTROL AND GIT

  • Adam Cribbs
  • March 21, 2017

In CGAT we are encouraged to contribute to lunch and learn sessions where we provide presentations on the 'ways in which we work in CGAT'. At one of these lunch and learn sessions I recently gave a presentation on version control (at the bottom of the page), specifically git, and thought that it would be useful to share some of the slides and information I shared during the presentation.

Version controlling is of central importance in software development. Any source code, even for small projects, are fairly complex documents. Often multiple persons work on the same code, or the same person might work on different parts of the same code. Without keeping track of changes and versions, any project would soon face collapse. Version control is a system to address these issues by:

  1. Being able to keep track of multiple versions of a collection documents (source code files).
  2. Merging different versions of a collection of documents.
  3. Keeping a history of all changes in order to reproduce the collection of documents at any point in time.

Version control software has been around for a number of years with a number of propriety and open source solutions. Previously popular solutions included CVS, subversion and mercurial. More recently, git has increased in popularity, driven by its ease of use, good support and easy to interact web portal github. This is the system that CGAT currently operate and have found it to be a really intuitive and easy way to share and contribute to CGAT code.

There are a number of really useful resources of information available for learning git and I have included them below in order of preference:
